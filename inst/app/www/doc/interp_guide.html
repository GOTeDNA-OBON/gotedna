Visual workflow summarizing how data is required to be formatted for integration with OBIS and/or GOTeDNA (i.e., user-uploaded data) (a) and how data is processed and analyzed within the GOTeDNA application (b).

Figure 1a summarizes the data workflow from field surveys, laboratory procedures, and bioinformatic pipelines, which are integrated and standardized to meet OBIS and FAIRe (Takahashi et al., 2025) requirements (see the GOTeDNA Standard Operating Procedure (SOP: https://github.com/GOTeDNA-OBON/gotedna, Appendix 4 for PDF)).

Figure 1b illustrates the backend of the GOTeDNA App, organized into several steps: selecting a data source; filtering data by criteria such as data type, taxon, and primers; defining a region using an interactive map; performing backend computations; choosing a detection threshold and protocol; and viewing panels that provide guidance on sampling windows, visualize eDNA detection probability and sampling effort, and display data authorship. Blue boxes represent the selection menus, pink boxes represent back-end computations, orange boxes represent information relating to the Map, and green boxes represent the outputs.
